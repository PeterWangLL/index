# 布局与内容架构 (Layout & Content Architecture)

> 本文档定义主页的页面结构、区块划分与各区块的职责。目前只关注"放什么板块"，不关注具体内容文案。

## 整体风格
- **单页滚动 (Single Page Scrolling)**：现代个人主页的标准形态，通过锚点导航跳转到不同区块。
- **视觉层级**：首屏抓眼 -> 关于我建立信任 -> 生活内容拉近距离 -> 技术与项目展示能力 -> 结尾引导联系。
- **动效节奏**：滚动即触发，不同区块有各自的性格动效（如 Life 区轻盈滑动、Tech 区利落线条、Reading 区柔和翻页感）。

---

## 页面区块规划

### 1. Navigation (固定顶部导航)
- **位置**: 页面顶部固定，z-index 最高。
- **样式**: 初始透明，滚动后变为玻璃态（backdrop-blur）+ 细底边。
- **内容**: 
  - Logo / 名字缩写
  - 锚点链接：Home、About、Life、Reading、Tech、Projects、Contact
  - 右侧：主题切换（ light / dark，可选，先预留）+ 一个有趣的交互入口（如命令面板按钮）。
- **动效**: 导航栏下滑入场，链接 hover 时磁吸或下划线展开效果。

### 2. Hero (首屏)
- **位置**: 100vh，视觉中心。
- **视觉**: 
  - 背景：极简深色/渐变，或低饱和度 Three.js 粒子/流体 shader（不抢文字风头）。
  - 中央：大字号名字 + 一句话 slogan（如 "Code. Hike. Read. Live."）。
  - 底部：向下滚动的 mouse scroll indicator。
- **动效**: 
  - 文字 stagger 入场（React Bits 文本效果）。
  - 背景粒子随鼠标轻微偏移（若使用 Three.js）。
  - 整体营造"呼吸感"。

### 3. About (关于我)
- **位置**: 首屏下方。
- **布局**: 左右分栏（左图右文，或上图下文在移动端）。
- **内容**: 
  - 一张个人头像/生活照（圆形或圆角矩形）。
  - 2-3 段简短自我介绍。
  - 底部：一排标签 pill（程序员、徒步爱好者、阅读者、摄影小白等）。
- **动效**: 滚动到该区域时，图片与文字从两侧向中间滑入汇合。

### 4. Life (生活记录)
- **位置**: About 之后。
- **定位**: 页面的"情绪核心"，用视觉卡片讲述生活故事。
- **布局**: 瀑布流或 Bento Grid（不规则网格），卡片大小不一。
- **内容类型**（每种对应一种卡片模板）:
  - **户外卡片**: 爬山/徒步，放风景照 + 地点 + 日期。
  - **活动卡片**: 参加展会/音乐会/聚会，放现场照片 + 活动名。
  - **运动卡片**: 小米运动数据截图，展示跑步/骑行记录（里程、配速、轨迹）。
- **动效**: 
  - 卡片 hover 时有 3D 倾斜或光泽扫过效果（Magic UI 3D Card）。
  - 滚动进入时卡片 stagger 弹出。

### 5. Reading (阅读记录)
- **位置**: Life 之后。
- **定位**: 营造"书架"氛围，展示阅读品味。
- **布局**: 水平滚动书架 或 3D 封面轮播（Carousel）。
- **内容**: 
  - 书籍封面（直立的矩形图片）。
  - 书名 + 作者。
  - 一句话/一段话读后感。
  - 评分（可选）。
- **动效**: 
  - 封面 hover 时微微放大、阴影加深，营造抽书感。
  - 轮播支持拖拽滚动（Framer Motion drag）。

### 6. Tech (技术 & 写作)
- **位置**: Reading 之后。
- **定位**: 展示技术成长，链接到现有博客文章。
- **布局**: 文章卡片列表（2-3 列网格）。
- **内容**: 
  - 最新/精选技术文章卡片：标题、摘要、标签、发布日期。
  - 一个"查看更多"按钮，链接到旧博客 `https://peterwangll.github.io/`。
- **动效**: 卡片干净利落的 slide-up 入场，hover 时边框高亮。

### 7. Projects (项目展示)
- **位置**: Tech 之后。
- **定位**: 展示个人作品，未来服务器部署项目的入口。
- **布局**: 项目网格（1-2 列大卡片）。
- **内容**: 
  - 每个项目：封面图、项目名、一句话描述、技术标签。
  - 链接状态：
    - 已部署：Live Demo + Source Code
    - 未部署：Coming Soon + Source Code
- **动效**: 卡片 hover 时图片缩放或出现渐变遮罩，信息上浮。

### 8. Contact / Footer (联系与页脚)
- **位置**: 页面最底部。
- **布局**: 简洁居中的 footer。
- **内容**: 
  - 社交图标链接：GitHub、Twitter/X、Email、Bilibili（可选）。
  - 一句结束语或引用。
  - 版权信息。
- **动效**: 链接 hover 时图标弹跳（Animate.css），或磁吸效果（Framer Motion）。

## 响应式断点策略
- **Desktop (≥1024px)**: 完整多栏布局，Three.js 背景开启，所有复杂动效生效。
- **Tablet (768px - 1023px)**: 减少栏数，简化部分动效。
- **Mobile (<768px)**: 单列堆叠，导航折叠为汉堡菜单，Three.js 背景可降级为静态渐变以优化性能。
