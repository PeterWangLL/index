# 布局与内容架构 (Layout & Content Architecture)

> 本文档定义主页的页面结构、区块划分与各区块的职责。参考 [qzq.at](https://qzq.at) 的叙事感主题分类风格，结合用户个人特点重新设计。

## 整体风格
- **单页滚动 (Single Page Scrolling)**：通过锚点导航跳转各主题区块。
- **叙事感 (Storytelling)**：每个区块不仅是罗列，更是在讲述"我是谁"、"我喜欢什么"。
- **视觉层级**：首屏抓眼 → 自我介绍建立信任 → 生活记录拉近距离 → 技术能力 → 兴趣爱好（Music / Books / Games） → 结尾引导联系。
- **动效节奏**：不同主题区有各自的性格动效（Life 区温暖呼吸、Code 区利落线条、Music 区轻盈流动、Books 区柔和沉稳、Games 区活泼弹跳）。

---

## 页面区块规划

### 1. Navigation (固定顶部导航)
- **位置**: 页面顶部固定，z-index 最高。
- **样式**: 初始透明，滚动后变为玻璃态（backdrop-blur）+ 细底边。
- **内容**: 
  - Logo / 名字缩写（Peter）
  - 锚点链接：Home、Introduction、Life、Code、Music、Books、Games、Contact
  - 右侧：主题切换入口（预留）或一个有趣的命令面板按钮。
- **动效**: 导航栏下滑入场，链接 hover 时磁吸或下划线展开效果。

### 2. Hero (首屏)
- **位置**: 100vh，视觉中心。
- **视觉**: 
  - 背景：极简深色/渐变，或低饱和度粒子/流体效果（不抢文字风头）。
  - 中央：大字号名字 **Peter Wang** + 一句话介绍（如 "在代码与生活的交界处，寻找自己的节奏。"）。
  - 底部：一排主题标签 pill（#programming #gaming #music #hiking）和向下滚动的指示器。
- **动效**: 
  - 文字 stagger 入场（Blur Text / Shiny Text）。
  - 背景有缓慢的呼吸感动画。

### 3. Introduction (自我介绍)
- **位置**: 首屏下方。
- **布局**: 左右分栏（左图右文，移动端上图下文）。
- **内容**: 
  - 一张个人头像/生活照。
  - 2-3 段简短自我介绍。
  - 底部标签 pill：程序员、孙燕姿歌迷、文明6玩家、微信读书用户、徒步爱好者。
- **动效**: 滚动到该区域时，图片与文字从两侧向中间滑入汇合。

### 4. Life (生活记录)
- **位置**: Introduction 之后。
- **定位**: 页面的"情绪核心"，记录生活中的美好瞬间。采用"主页精选 + 子频道可扩展"的架构。
- **主页内容策略**: 只展示最新/最精选的 4-6 张卡片，不堆叠全部内容。
- **子板块架构**（未来可持续扩展，当前在主页用分类标签或分组占位体现）:
  1. **Fitness (健身)**: 小米运动数据、跑步轨迹、健身房记录。
  2. **Travel (旅游)**: 城市风光、旅行见闻、打卡记录。
  3. **Hiking (爬山/徒步)**: 登顶照片、徒步路线、山间风景。
  4. **Moments (重要时刻)**: 毕业典礼、过年回家的高铁、生日聚会、有纪念意义的瞬间。
  5. **Photography (随拍)**: 生活中发现的美，街拍、日落、花草、城市角落。
- **布局**: 
  - 顶部：区块标题 + 一句话描述。
  - 中部：子板块标签栏（Fitness / Travel / Hiking / Moments / Photography），点击可筛选主页精选内容。
  - 下方：Bento Grid 精选卡片网格，支持 `object-cover` 裁剪，按图片形状分配 `col-span`。
- **动效**: 卡片 hover 时图片微微放大；底部渐变遮罩上显示标题和描述；切换子板块时内容自动过滤。

### 5. Code (代码与技术)
- **位置**: Life 之后。
- **定位**: 展示硬核技术文章与项目。文章为站内 Markdown 渲染，不链接到外部旧博客；项目区域暂时隐藏，待 NAS 部署完成后再展示。
- **内容架构**:
  - **文章列表**：精选技术文章卡片，点击后进入站内文章详情页（基于 Markdown 渲染）。
  - **文章页路由**：`/blog/[slug]`，使用 Next.js Static Export 生成静态文章页。
- **布局**: 文章卡片网格（2-3 列），每张卡片显示标题、摘要、标签、发布日期。
- **动效**: 卡片利落 slide-up 入场；hover 时边框高亮；Terminal 组件可作为装饰元素展示技术栈。

### 6. Music (音乐)
- **位置**: Code 之后。
- **定位**: 以**时间线**的形式记录"这段时间在听什么"。不是展示专辑收藏，而是记录某个时期反复播放的歌曲，以及当时的心情和感想。
- **主页内容策略**: 展示精选歌曲时间线卡片（包含时期、歌名、歌手、简短感想）。
- **详情交互**: 每条时间线卡片支持展开/收起，显示更长的听后感。
- **布局**: 垂直时间线，左侧时间轴，右侧卡片内容。
- **内容示例**:
  - 2025.04 — 王力宏《爱错》
  - 2025.02 — 孙燕姿《天黑黑》
  - 2024.11 — 陶喆《沙漏》
- **动效**: 滚动进入时时间线 stagger 入场；卡片 hover 时边框高亮；展开/收起有平滑高度过渡。

### 7. Books (阅读)
- **位置**: Music 之后。
- **定位**: 展示阅读积累与习惯，强调微信读书。主页放精选书籍，每本书支持点击进入**详情页**阅读完整读后感。
- **主页内容策略**: 
  - 左侧：数据亮点卡片（总阅读时长、读完的书数量、连续听书天数）。
  - 右侧：精选书籍横向滚动列表，每本书有封面、书名、作者、一句话摘要和"阅读笔记"入口。
- **详情页路由**: `/books/[slug]`，静态生成，展示书籍信息和完整读后感。
- **动效**: 数据数字 Count Up 动画；书籍卡片 hover 时封面微微放大。

### 8. Games (游戏)
- **位置**: Books 之后。
- **定位**: 展示游戏爱好，活泼有趣。
- **布局**: Bento Grid 或特色卡片网格。
- **内容**: 
  - **炉石传说：酒馆战棋**：战绩截图/段位/ favorite 英雄。
  - **哈迪斯 (Hades)**：游戏截图/通关次数/ favorite 武器。
  - **文明6**：刚入坑的新手宣言/ favorite 领袖/ "再打一回合就睡觉"梗。
- **动效**: 卡片 hover 时有 3D 倾斜或弹性反馈（Bounce Cards），营造游戏感。

### 9. Contact (联系与页脚)
- **位置**: 页面最底部。
- **定位**: 以一个简洁但富有仪式感的收尾与访客建立连接。参考 [qzq.at](https://qzq.at) 的风格：一句个人座右铭/结语，配合文字动效，下方是社交图标。
- **布局**: 
  - 中央：大字号引用式文字（如 *"There are countless ways to tell a story about the world, this is mine."*）。
  - 动效：滚动进入视野时，文字逐词淡入（word-by-word stagger reveal），营造叙事的尾声感。
  - 下方：一排圆形社交图标按钮（GitHub、Email 等），hover 时有背景填充或轻微放大效果。
  - 最底部：极简版权信息。

## 响应式断点策略
- **Desktop (≥1024px)**: 完整多栏布局，复杂动效生效，横向滚动区展现最佳效果。
- **Tablet (768px - 1023px)**: 减少栏数，Bento Grid 调整为 2 列，简化部分动效。
- **Mobile (<768px)**: 单列堆叠，导航折叠为汉堡菜单，横向滚动区保持横向滑动手势。
