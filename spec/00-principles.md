# 核心原则与项目记忆

> 本文档记录项目的核心定位、不变的原则与长期记忆。无论上下文如何切换，这些约束始终生效。

## 项目定位
- **项目名**: `index`
- **性质**: 个人主页（Personal Homepage / Portfolio）
- **目标**: 向外界展示「我是谁」，不仅是技术成长，更是生活态度与经历。
- **愿景**: 未来作为所有个人项目的导航中枢（Hub），服务器部署后可跳转至各个子项目。

## 域名与部署
- **已有域名**: `www.peterwangll.top`
- **部署计划**: 
  1. 项目开发阶段在本地进行。
  2. 完成后部署到 GitHub Pages。
  3. 绑定自定义域名 `www.peterwangll.top`，替代现有博客 `https://peterwangll.github.io/`。
- **配置约定**: 因最终绑定自定义域名，Next.js `basePath` 保持默认（不配置）。

## 内容定位
内容分为两大主轴：
1. **技术成长 (Tech)**: 博文、项目、开源贡献、学习笔记。
2. **生活记录 (Life)**: 
   - 户外活动（爬山、徒步、旅行）
   - 运动数据（小米运动截图等）
   - 阅读记录（书籍封面 + 简短读后感）
   - 参加活动、展览、聚会等

## 设计哲学
- **个性化**: 拒绝千篇一律的预制主题，每一个细节都要亲手打磨。
- **动效丰富**: 使用现代 React 动效库打造沉浸式浏览体验。
- **生活化**: 界面氛围要温暖、有呼吸感，不像冷冰冰的企业官网。
- **细节控**: 微交互、过渡动画、排版节奏、颜色搭配都需要反复推敲。
- **克制堆叠**: 动效组件需精挑细选，每个效果都应有其存在理由，禁止为了炫技而盲目堆砌。

## 工作模式
- **Spec Coding**: 每次修改前必须先查看 `spec/` 目录下的相关文档，确保理解当前上下文；修改后必须同步更新对应的 spec 文档，保证文档永远反映最新状态。
- **渐进式迭代**: 先搭架子（布局、路由、技术栈），再填内容，最后精修动效与细节。
